     Context
    
     作为运维开发人员，我们需要开发一款 Go 语言编写的 CLI 工具，用于解决以下问题：
    
     - 问题场景：客户集群环境中经常缺少基本的运维工具（jq、curl、telnet 等）
     - 网络限制：部分客户环境没有网络连接，无法在线安装工具
     - 使用便捷性：需要一个单一的二进制文件，开箱即用
    
     ---
     1. 产品概述
    
     1.1 产品定位
    
     OpsKit 是一款单一二进制、零依赖、内置常用运维工具的 CLI 工具箱，专为受限环境下的故障排查设计。
    
     1.2 解决的核心问题
    
     ┌──────────────────────┬──────────────────────────┐
     │         问题         │         解决方案         │
     ├──────────────────────┼──────────────────────────┤
     │ 客户集群缺少基础工具 │ 内置常用工具，无需安装   │
     ├──────────────────────┼──────────────────────────┤
     │ 环境无网络连接       │ 单文件分发，无需联网下载 │
     ├──────────────────────┼──────────────────────────┤
     │ 工具版本不一致       │ 统一版本，行为可预测     │
     ├──────────────────────┼──────────────────────────┤
     │ 权限限制无法安装     │ 静态编译，可直接执行     │
     └──────────────────────┴──────────────────────────┘
    
     1.3 目标用户
    
     - SRE/运维工程师
     - DevOps 工程师
     - 技术支持工程师
     - 需要在客户现场排查问题的工程师
    
     1.4 设计原则
    
     ┌────────────┬─────────────────────────────────────────┐
     │    原则    │                  说明                   │
     ├────────────┼─────────────────────────────────────────┤

     ├────────────┼─────────────────────────────────────────┤

     2. 核心架构：内嵌二进制方案
    
     2.1 方案说明
    
     利用 Go 1.16+ 的 //go:embed 指令，在编译阶段将各平台的原生工具二进制文件作为静态资源打包进 opskit 可执行文件。
    
     构建阶段：
       jq_linux_amd64  ──┐
       curl_linux_amd64──┤  go:embed  ──►  opskit（单一二进制）
       busybox_amd64   ──┘
    
     运行阶段：
       opskit jq '.foo' data.json
            │
            ▼
       检测当前 GOOS/GOARCH
            │
            ▼
       释放对应二进制到 ~/.cache/opskit/<version>/jq
            │
            ▼
       chmod +x，exec.Command 调用，参数透传
            │
            ▼
       输出结果给用户（stdout/stderr 直接透传）
    
     2.2 释放策略
    
     - 释放目录优先级：
       a. $OPSKIT_BIN_DIR/<version>/（用户自定义）
       b. $HOME/.cache/opskit/<version>/（推荐）
       c. /tmp/.opskit-bin-<version>/（fallback）
     - 懒释放：首次调用某工具时才释放，不是启动时全量释放
     - 幂等：若文件已存在且 SHA256 一致，跳过释放，直接执行
     - 清理：提供 opskit clean 命令清理临时目录；进程退出时不主动删除（便于复用）
     - 权限：释放后 chmod 0755，确保可执行

     3. 功能需求
    
     3.1 工具代理调用（核心）
    
     - opskit <tool> [args...] — 完整透传所有参数给内嵌工具
     - stdout / stderr / 退出码全部透传，与直接调用原生工具行为完全相同
     - 支持管道：cat data.json | opskit jq '.name'
     - 支持信号透传（Ctrl+C 正常终止）
    
     3.2 工具管理命令
    
     ┌─────────────────────────────┬──────────────────────────────────────────┐
     │            命令             │                   说明                   │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit list                 │ 列出所有内置工具及其版本、架构           │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit version              │ 输出 opskit 自身版本及构建信息           │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit extract <tool>       │ 将指定工具释放到当前目录（便于单独使用） │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit extract --all        │ 释放所有工具到当前目录                   │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit extract --dir <path> │ 指定释放目录                             │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit clean                │ 清理临时释放目录                         │
     ├─────────────────────────────┼──────────────────────────────────────────┤
     │ opskit which <tool>         │ 显示工具的临时释放路径                   │
     └─────────────────────────────┴──────────────────────────────────────────┘
    
     3.3 全局选项
    
     ┌──────────────────┬──────────────────────────┐
     │       选项       │           说明           │
     ├──────────────────┼──────────────────────────┤
     │ --verbose / -v   │ 输出 opskit 内部调试信息 │
     ├──────────────────┼──────────────────────────┤
     │ --bin-dir <path> │ 覆盖临时释放目录         │
     ├──────────────────┼──────────────────────────┤
     │ --version        │ 输出版本信息             │
     └──────────────────┴──────────────────────────┘
    
     3.4 命令设计示例
    
     # 直接使用内置工具
     opskit jq '.name' data.json
     opskit curl https://example.com
     opskit telnet example.com 80
     opskit ping example.com
    
     # 支持管道
     kubectl logs pod-xxx | opskit jq '.message'
    
     # 管理命令
     opskit list
     opskit extract jq
     opskit clean

     5. 非功能需求
    
     5.1 性能要求
    
     - 启动时间（首次，含释放）：< 500ms
     - 启动时间（后续，文件已存在）：< 50ms
     - 单二进制文件大小：< 50MB（理想 < 20MB，可使用 UPX 压缩）
     - 内存占用：< 100MB（常规使用）
    
     5.2 兼容性要求
    
     ┌─────────┬──────────────┬──────────────────────────┐
     │  平台   │     架构     │         最低版本         │
     ├─────────┼──────────────┼──────────────────────────┤
     │ Linux   │ amd64, arm64 │ CentOS 7+, Ubuntu 18.04+ │
     ├─────────┼──────────────┼──────────────────────────┤
     │ Linux   │ 386, armv6/7 │ 按需支持                 │
     ├─────────┼──────────────┼──────────────────────────┤
     │ macOS   │ amd64, arm64 │ 10.15+（可选）           │
     ├─────────┼──────────────┼──────────────────────────┤
     │ Windows │ amd64        │ Windows 10+（可选）      │
     └─────────┴──────────────┴──────────────────────────┘
    
     5.3 安全性要求
    
     - 构建时验证所有工具的 SHA256，防止供应链污染
     - 释放前校验 SHA256，防止临时目录中的文件被篡改
     - tools.yaml 中明确记录工具来源与哈希，可审计
     - Release 二进制本身提供 SHA256 校验文件

     6.4 核心代码示例
    
     // internal/embed/manager.go
     package embed
    
     import (
         "embed"
         "os"
         "path/filepath"
         "runtime"
     )
    
     //go:embed assets/*
     var assets embed.FS
    
     type BinaryManager struct {
         cacheDir string
     }
    
     func (m *BinaryManager) GetPath(toolName string) (string, error) {
         // 1. 检查缓存是否已存在
         cachedPath := filepath.Join(m.cacheDir, toolName)
         if m.checkCache(cachedPath, toolName) {
             return cachedPath, nil
         }
    
         // 2. 从嵌入文件解压到缓存
         assetPath := m.getAssetPath(toolName)
         data, err := assets.ReadFile(assetPath)
         if err != nil {
             return "", err
         }
    
         // 3. 写入缓存 + 加执行权限
         err = os.WriteFile(cachedPath, data, 0755)
         if err != nil {
             return "", err
         }
    
         return cachedPath, nil
     }
    
     func (m *BinaryManager) getAssetPath(toolName string) string {
         return fmt.Sprintf("assets/%s-%s/%s",
             runtime.GOOS, runtime.GOARCH, toolName)
     }
    
     // internal/builtins/jq.go
     package builtins
    
     import (
         "os"
         "os/exec"
         "syscall"
         "opskit/internal/embed"
     )
    
     func RunJQ(mgr *embed.BinaryManager) error {
         binPath, err := mgr.GetPath("jq")
         if err != nil {
             return err
         }
    
         // 透传所有参数: opskit jq ... -> jq ...
         cmd := exec.Command(binPath, os.Args[2:]...)
         cmd.Stdin = os.Stdin
         cmd.Stdout = os.Stdout
         cmd.Stderr = os.Stderr
    
         // 透传退出码
         if err := cmd.Run(); err != nil {
             if exitErr, ok := err.(*exec.ExitError); ok {
                 if status, ok := exitErr.Sys().(syscall.WaitStatus); ok {
                     os.Exit(status.ExitStatus())
                 }
             }
             return err
         }
         return nil
     }

     8. 构建流程
    
     8.1 Makefile 示例
    
     VERSION ?= $(shell git describe --tags --always --dirty)
    
     # 下载第三方二进制
     download-tools:
        ./scripts/download-tools.sh
    
     # 构建当前平台
     build: download-tools
        CGO_ENABLED=0 go build -ldflags="-s -w -X main.Version=$(VERSION)" \
                -o bin/opskit ./cmd/opskit
    
     # 构建所有平台
     build-all: download-tools
        goreleaser build --snapshot
    
     # 运行测试
     test:
        go test -v ./...
    
     # 清理
     clean:
        rm -rf bin/ dist/ ~/.cache/opskit/
    
     .PHONY: download-tools build build-all test clean
    
     8.2 完整构建步骤
    
     # 步骤 1：根据 tools.yaml 下载各平台工具二进制并校验 SHA256
     ./scripts/download-tools.sh
    
     # 步骤 2：编译 opskit（embed 会自动打包 assets/ 目录）
     CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
       go build -ldflags="-s -w -X main.Version=v1.0.0" \
       -o dist/opskit_linux_amd64 ./cmd/opskit
    
     # 步骤 3：可选 UPX 压缩
     upx --best dist/opskit_linux_amd64
    
     # 步骤 4：生成 SHA256
     sha256sum dist/opskit_linux_amd64 > dist/opskit_linux_amd64.sha256

     9. 使用场景
    
     场景 1：JSON 日志分析
    
     问题：需要分析容器日志，但环境没有 jq。
    
     # 使用 opskit
     kubectl logs pod-xxx | opskit jq '.message'
    
     场景 2：服务连通性测试
    
     问题：需要验证服务是否正常监听，但没有 telnet/curl。
    
     # 测试 TCP 端口
     opskit telnet db-service 5432
    
     # 测试 HTTP 接口
     opskit curl http://api-service/health
    
     场景 3：配置文件检查
    
     问题：需要检查 YAML 配置，但环境没有 yq。
    
     opskit yq eval '.spec.replicas' deployment.yaml
    
     场景 4：离线安装使用
    
     # 只需要分发一个文件
     scp opskit user@customer-host:/tmp/
     ssh user@customer-host
     chmod +x /tmp/opskit
     /tmp/opskit jq --help
